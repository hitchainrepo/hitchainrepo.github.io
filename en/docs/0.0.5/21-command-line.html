Create hit configuration

At first using hit you need to create hit configuration: hit cfg create 


$> hit cfg create
!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!
Please write down the mnemonic words(for recover your password)!!! :
alert maze mimic half fat depth motor trigger tape that ribbon also
!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!!
Hit config is updated.
When you create hit configuration, you need a password and generate a password mnemonic for password retrieval.

hit password recover

Recover by password mnemonic.

  • hit password recover command

    • hit cfg recover password|account-pri-key|rsa-pri-key

  • Recover Password

    • hit cfg recover password
    Input: password mnemonic
    Parameters:

  • Recover Private Key

    • hit cfg recover account-pri-key
    Parameters:

  • Recover RSA Private Key

    • hit cfg recover rsa-pri-key
    Parameters:

    hit Account management

    hit can management multiple accounts you can switch to different account. This account use to contact with smart contract.

  • hit Account Commands

    • hit cfg account [add|remove] name [priKey] [pubKey] password
    hit cfg account set name

  • Create Account

    • hit cfg account add name [priKey] [pubKey] password
    Parameters:
    name: [Required], Account name, aA-zZ and numbers
    priKey, pubKey: [Option], private key and public key is require when adding exists account
    password: [Required], the password of hit cfg

  • Delete account

    • hit cfg account remove name
    Parameters:
    name: [Required], account name

  • Setting default account

    • hit cfg account set name
    Parameters:
    name: [Required], account name

  • List add accounts

    • hit cfg account

    Smart contract GAS config

    hit can maintain multiple smart contract GAS config, you can switch to different config. This GAS config use to invoke smart contract, including the smart contract deploy and write.

  • GAS config Commands

    • hit cfg gas add name deployGas deployGwei writeGas writeGwei
    hit cfg gas remove name
    hit cfg gas set name

  • Create GAS config

    • hit cfg gas add name deployGas deployGwei writeGas writeGwei
    Parameters:
    name: [Required], The GAS config name, aA-zZ and numbers
    deployGas: [Required], the GAS for contract deploy
    deployGwei: [Required], the GWei for contract deploy
    writeGas: [Required], the GAS for contract write
    writeGwei: [Required], the GWei for contract write

  • Delete GAS config

    • hit cfg gas remove name
    Parameters:
    name: [Required], the GAS config name

  • Setting default GAS config

    • hit cfg gas set name
    Parameters:
    name: [Required], the name of GAS config

  • List all GAS configs

    • hit cfg gas

    • hit am pathId
    Parameters:
    pathId: [Required], PullRequest id
    --ignore-space-change: [Option], Merge patch ignore space
    --ignore-white-sapce: [Option], Merge patch ignore white space
    --force-merge: [Option], force merge content overwrite line content
    --no-commit: [Option], if this options is not specify will keep all commits, otherwise not keep original commits
