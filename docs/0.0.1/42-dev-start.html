模块描述

hit 在 jgit 项目模块的基础上添加两个模块：

  • org.hitchain.core hit的主要核心代码
  • org.hitchain.core.test hit用于测试代码

    • 模块间设计

    为了尽量减少对原有的 jgit 源代码的改动，hit 功能主要采用反射注入的方式进行功能植入。 对于 jgit 的代码修改部分，采用特定的标识/*add by hit*//*end of add*/， 可以整个项目进行搜索代码标识，就可以知到对 jgit 源码修改的部分。
